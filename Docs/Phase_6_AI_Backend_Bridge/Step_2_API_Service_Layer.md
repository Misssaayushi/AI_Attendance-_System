# Step 2: API Service Layer

## Objective
Create `ai_module/api_service.py` as a reusable HTTP client manager for verified attendance events.

## Main Responsibility
`api_service.py` receives a verified attendance event from the AI module, sends it to the backend, and returns a small response object that the recognition loop can display safely.

## Tasks
1. Create `AttendancePayload` type hints.
   - `student_id: str`
   - `name: str`
   - `confidence: float`
   - `timestamp: str`
   - `status: str`

2. Create `AttendanceAPIResponse` type hints.
   - `success: bool`
   - `message: str`
   - `status_code: int | None`
   - `mocked: bool`
   - `error: str | None`

3. Create `AttendanceAPIService`.
   - Initialize with URL, timeout, retry count, and mock mode values from `config.py`.
   - Use `get_logger("API_Connector")`.
   - Use the `requests` library for HTTP POST calls.

4. Implement payload creation.
   - Timestamp must be generated by the AI module.
   - Confidence must be sent as the AI-calculated confidence score.
   - Status should default to `Present`.

5. Implement transmission logic.
   - Send `POST` request to `/api/v1/attendance/verify`.
   - Accept `200 OK` and `201 Created` as success.
   - Treat other status codes as backend failure.

6. Implement graceful error recovery.
   - Handle connection errors.
   - Handle timeout errors.
   - Handle invalid JSON responses.
   - Return informative messages instead of raising exceptions into the webcam loop.

7. Implement simple retry behavior.
   - Retry only for network-level failures or timeout errors.
   - Keep retry count low to avoid freezing the video feed.

## Expected Outcome
`api_service.py` provides a plug-and-play API client that can be called from `recognize_faces.py` without exposing the recognition loop to network exceptions.
